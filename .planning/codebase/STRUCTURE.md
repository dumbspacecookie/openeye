## Directory Purposes

**src/** - TypeScript Agent Layer
- Purpose: Runs pi-agent decision loop, manages tool calls, exposes public API
- Contains: Agent class, HTTP client, tool definitions, model configs, memory compat layer
- Key files: `index.ts` (main), `sidecar-client.ts` (subprocess bridge), `tools.ts` (8 tools)

**sidecar/** - Python Persistence Layer
- Purpose: FastAPI server running in subprocess, manages SQLite state, handles persistence and search
- Contains: HTTP endpoints (20+), state management, skill loading, trajectory export, cloud sync
- Key files: `server.py` (FastAPI app), `state.py` (SQLite schema + queries), `skills.py` (skill retrieval)

**skills/** - Procedural Memory (Markdown)
- Purpose: Domain-specific step verification criteria injected into agent system prompt
- Contains: Markdown files with YAML front-matter, grouped by domain
- Key files: `medical/hand-hygiene.md` (5-step protocol), `manufacturing/visual-qc.md`, `field-service/lockout-tagout.md`
- Domains: medical, manufacturing, field-service, general

**schemas/** - JSON Schema
- Purpose: Type definitions for skill front-matter validation
- Contains: JSON Schema files for skill metadata validation

**eval/** - Benchmark
- Purpose: Deterministic 100-example evaluation dataset
- Contains: Test examples with expected outputs, evaluation scripts
- Key files: `eval-data/` (examples), `scripts/` (runners)

**tests/** - Test Suites
- Purpose: Ensure agent/client/sidecar functionality
- Contains: TypeScript tests (vitest) and Python tests (pytest)
- Key files: `runner.test.ts` (integration), `test_sidecar.py` (pytest)

**dist/** - Compiled Output
- Purpose: Generated by `npm run build` (tsc)
- Contains: .js and .d.ts files from src/ TypeScript
- Committed: No (ignored by .gitignore)

**.planning/codebase/** - Planning Documents
- Purpose: GSD mapping documents (this directory)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, etc.
- Generated: Yes (created by /gsd-map-codebase)

## Key File Locations

**Entry Points:**

- `src/index.ts`: Main TypeScript entry point. Exports OpenEyeAgent class, model constants, createOpenEyeTools, Memory class, setupProviders/makeStreamFn
- `sidecar/server.py`: Python sidecar entry. FastAPI app started via `python -m uvicorn server:app --port 7770`
- `package.json`: npm package config, defines main/exports fields pointing to dist/index.js

**Agent Implementation:**

- `src/index.ts`: OpenEyeAgent class (create, prompt, captureAndClose, exportTrajectories, pushToHub, stop)
- `src/sidecar-client.ts`: SidecarClient class managing subprocess + HTTP bridge (30+ methods for sessions, frames, steps, skills, trajectories)
- `src/tools.ts`: 8 AgentTool objects registered on pi-agent (search_memory, search_frames, recall_skill, write_skill, start_visual_session, end_visual_session, log_frame, verify_step)

**Models & Configuration:**

- `src/models.ts`: 30+ LLM model exports (Anthropic, OpenAI, Google, Groq, Mistral, OpenRouter, Bedrock, Ollama, custom)
- `package.json`: Exports map for modular imports (`./memory`, `./models`, `./tools`)

**Sidecar Implementation:**

- `sidecar/server.py`: FastAPI endpoints (20+), request models (Pydantic), lifespan manager
- `sidecar/state.py`: SQLite state engine, schema (12 tables), FTS5 indexing, thread-safe accessors (~650 lines)
- `sidecar/skills.py`: Skill discovery, YAML parsing, semantic skill matching
- `sidecar/trajectories.py`: Trajectory creation, ShareGPT format export
- `sidecar/cloud_sync.py`: Async worker for pushing sync_pending rows to cloud endpoint

**Skills (Procedural Memory):**

- `skills/medical/hand-hygiene.md`: 5-step clinical protocol with pass/fail/uncertain criteria
- `skills/medical/surgical-site-prep.md`: Pre-surgical verification protocol
- `skills/manufacturing/equipment-pre-op-check.md`: Equipment safety checklist
- `skills/manufacturing/visual-qc.md`: Visual quality control criteria
- `skills/field-service/lockout-tagout.md`: LOTO compliance protocol

**Testing:**

- `tests/runner.test.ts`: Integration test for OpenEyeAgent lifecycle (create → prompt → export → close)
- `tests/preflight.test.ts`: Environment validation (Python availability, sidecar paths)
- `tests/test_sidecar.py`: State engine tests, skill loading, trajectory export

**Configuration:**

- `tsconfig.json`: TypeScript compiler options (ESM, strict, lib dom+es2022)
- `vitest.config.ts`: Vitest runner config
- `package.json`: Scripts (build, test, typecheck), dependencies, exports

## Naming Conventions

**Files:**

- TypeScript: `camelCase.ts` (index.ts, sidecar-client.ts, preflight.ts)
- Python: `snake_case.py` (server.py, state.py, skills.py, cloud_sync.py)
- Skill markdown: `kebab-case.md` (hand-hygiene.md, equipment-pre-op-check.md, lockout-tagout.md)
- Config: lowercase + dots (tsconfig.json, vitest.config.ts, package.json)

**Directories:**

- Source: lowercase (src, sidecar, skills, tests, dist, schemas, eval)
- Nested: lowercase by domain (skills/medical, skills/manufacturing, eval/eval-data, eval/scripts)

**Functions/Classes:**

- TypeScript: PascalCase for classes (OpenEyeAgent, SidecarClient, Memory), camelCase for functions (createOpenEyeTools, makeStreamFn, setupProviders)
- Python: snake_case for functions (get_db, create_session, search_fts5_messages), SCREAMING_SNAKE_CASE for constants (OPENEYE_HOME, DB_PATH, SCHEMA_VERSION)

**Exports:**

- Public exports from main (src/index.ts): OpenEyeAgent, SidecarClient, Memory, model constants (ANTHROPIC_SONNET, etc.), helper functions (setupProviders, makeStreamFn, createOpenEyeTools)
- Modular exports: ./memory (Memory class), ./models (model constants), ./tools (createOpenEyeTools)

## Where to Add New Code

**New LLM Provider:**
- Add to `src/models.ts` following pattern of existing providers (export const PROVIDER_MODEL = getModel("provider", "model-id"))
- Ensure environment variable documented in README.md
- Test by importing and creating agent with new model

**New Tool:**
- Add AgentTool object to `src/tools.ts` (name, label, description, parameters Type schema, async execute handler)
- Handler calls appropriate SidecarClient method
- Register tool in createOpenEyeTools function's return array

**New Skill Domain:**
- Create subdirectory in `skills/` (e.g., skills/aviation/)
- Add markdown files with YAML front-matter: name (kebab-case), domain, description, device_types, reward_threshold
- Include pass/fail/uncertain criteria for each procedure step
- Validate with `python sidecar/validate_skill.py skills/aviation/`

**New Sidecar Endpoint:**
- Add request model (Pydantic BaseModel) in `sidecar/server.py`
- Add @app.post or @app.get handler function
- Call underlying state.py or skills.py functions
- Return JSON response with appropriate status codes
- Test with `python -m pytest tests/test_sidecar.py::test_new_endpoint`

**New Agent Test:**
- Create file in `tests/` (e.g., tests/export.test.ts)
- Import { describe, it, expect } from vitest
- Use OpenEyeAgent.create with test model
- Test with `npm test`

**New Python Test:**
- Create file in `tests/` (e.g., tests/test_new_feature.py)
- Import pytest, state, skills modules
- Use fixtures for test DB (tmpdir)
- Run with `python -m pytest tests/test_new_feature.py -v`

## Package Structure & Exports

**Main export** (`src/index.ts` → `dist/index.js`):
```typescript
export { OpenEyeAgent };
export { SidecarClient };
export { createOpenEyeTools };
export { ANTHROPIC_OPUS, ANTHROPIC_SONNET, ANTHROPIC_HAIKU };
export { setupProviders, makeStreamFn };
```

**Modular exports** (package.json exports field):
- `"."`: main index.js
- `"./memory"`: memory.js (Memory class)
- `"./models"`: models.js (model constants)
- `"./tools"`: tools.js (createOpenEyeTools)

**Import patterns:**
```typescript
// Main API
import { OpenEyeAgent, setupProviders, makeStreamFn, ANTHROPIC_SONNET } from "@openeye/pi-openeye";

// Memory compat layer
import { Memory } from "@openeye/pi-openeye/memory";

// Model constants
import { ANTHROPIC_OPUS, OPENAI_GPT41, GROQ_LLAMA33_70B } from "@openeye/pi-openeye";

// Tools function
import { createOpenEyeTools } from "@openeye/pi-openeye";
```
